Informe: Visualización en Pantalla LCD con Caracteres Personalizados

1. Introducción

Este informe describe un proyecto con Arduino cuyo objetivo es controlar una pantalla de cristal líquido (LCD) de 16x2. La actividad se centra en mostrar texto específico en posiciones determinadas y en diseñar y visualizar cuatro caracteres personalizados, uno en cada esquina de la pantalla.

2. Componentes Utilizados

  • Placa Arduino Uno
  • Pantalla LCD 16x2
  • Potenciómetro (para ajustar el contraste de la pantalla)
  • Resistencia
  • Placa de pruebas (Protoboard)
  • Cables de conexión

3. Funcionamiento y Características Clave

El funcionamiento del proyecto se basa en la librería LiquidCrystal de Arduino, que facilita enormemente la comunicación con la pantalla LCD.

Las características principales del código son:

  • Creación de Caracteres Personalizados:
    Se han diseñado cuatro íconos únicos (sorpresa, música, corazón y campana) mediante arreglos de bytes. Cada byte representa una fila de 8 píxeles (5x8 en total por carácter). Estos diseños se cargan en la memoria de la LCD al inicio con la función lcd.createChar().
  • Posicionamiento en Pantalla:
    Se utiliza el comando lcd.setCursor(columna, fila) para mover el cursor a una posición exacta antes de escribir. Esto permite colocar los caracteres personalizados en las cuatro esquinas de la pantalla (posiciones [0,0], [15,0], [0,1] y [15,1]).
  • Visualización de Texto:
    De manera similar, se posiciona el cursor para escribir el nombre "Josue" en la primera fila y el apellido "Jacome" en la segunda. Aunque el código intenta centrar el texto manualmente ajustando la columna de inicio, una técnica más precisa podría calcular la posición de inicio basándose en la longitud del texto.

A continuación se presenta la simulación interactiva del proyecto, donde se puede ver el resultado final en la pantalla LCD tal como se ha programado.

5. Conclusión

Esta actividad ha sido muy útil para aprender a manejar pantallas LCD, algo fundamental en muchos proyectos para crear interfaces de usuario. Se ha comprendido cómo posicionar texto y, lo más interesante, cómo diseñar y mostrar gráficos o símbolos personalizados.

Segunda Actividad


1. Introducción

Este informe detalla un sistema integrado con Arduino que mide la distancia con un sensor ultrasónico, muestra esta información en tiempo real en una pantalla LCD y, además, activa una señal visual (un LED) o una alarma sonora (un zumbador) dependiendo de si la distancia supera un umbral .

2. Componentes Utilizados

  • Placa Arduino Uno
  • Pantalla LCD 16x2
  • Sensor Ultrasónico de Distancia
  • LED
  • Alarma
  • Resistencias
  • Placa de pruebas (Protoboard)
  • Cables de conexión

3. Funcionamiento y Características Clave

Este proyecto combina la lectura de un sensor con múltiples salidas (visual y sonora) y la visualización de datos. El código está estructurado para ser eficiente y claro.

  • Medición de Distancia:
    El código utiliza una función llamada obtenerDistancia(). Esta función realiza todo el proceso de enviar un pulso ultrasónico desde el pin 8 y medir el tiempo del eco para calcular la distancia en centímetros.
  • Lógica Condicional (Foco y Alarma):
    En el bucle principal (loop), el programa primero obtiene la distancia. Luego, un bloque if-else decide qué acción tomar:
    • Si la distancia es mayor o igual a 100 cm, se enciende el foco LED (conectado al pin 9) y se asegura de que la alarma (en el pin 10) esté apagada.
    • Si la distancia es menor a 100 cm, se apaga el foco LED y se enciende la alarma para alertar sobre la proximidad de un objeto.
  • Visualización en Pantalla LCD:
    Independientemente de la distancia, la pantalla LCD se actualiza constantemente para mostrar información útil al usuario. En la primera fila se muestra el texto fijo "Distancia". En la segunda fila se muestra el valor numérico actual, por ejemplo: "Valor = 50".
  • Control de Refresco de Pantalla:
    El código incluye una variable contadorClear. Su propósito es limpiar la pantalla (lcd.clear()) solo cada 5 ciclos de lecturas seguras (distancia > 100). Esto es para evitar el parpadeo constante y que limpie toda la pantalla y no se aprecie.

4. Simulación del Circuito en Tinkercad

A continuación se presenta la simulación interactiva de este proyecto integrado. Se puede mover el objeto para cambiar la distancia y observar en tiempo real cómo cambian los valores en la pantalla LCD y cómo se activan el foco LED o la alarma sonora según corresponda.

5. Conclusión

Este proyecto es un ejemplo de un sistema electrónico funcional. Se ha logrado integrar exitosamente un sensor para capturar datos del entorno, una pantalla para ofrecer una interfaz visual al usuario y actuadores (luz y sonido) que reaccionan según una lógica .
